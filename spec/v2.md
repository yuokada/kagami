# Shadow Proxy Specification v2.1

# 1. 概要

Shadow Proxy は **既存 API と開発中 API のレスポンス互換性を検証するための開発ツール**である。

本サーバーは既存 API の **drop-in replacement** として動作し、
同一リクエストを以下の2つの API に送信する。

- Existing API Server（master）
- Development API Server（shadow）

両者のレスポンスを比較し、差分を検出した場合は **標準出力にレポートする**。

---

# 2. ペルソナ

主要ユーザー

| ペルソナ | 説明 |
|---|---|
| API Developer | API互換性を検証する開発者 |

目的

- API migration の検証
- リファクタリング後の互換性確認
- 新API実装の挙動確認

---

# 3. 利用フェーズ

想定フェーズ

| フェーズ | 利用 |
|---|---|
| 開発初期 | ✔ |
| 開発中 | ✔ |
| QA | optional |
| ユーザー検証 | ✘ |
| 本番 | ✘ |

理由

- 比較処理による遅延が発生する
- 可用性・セキュリティを考慮していない

# 4. アーキテクチャ

Developer Client
        |
        v
   Shadow Proxy
      /     \
     v       v
Existing API  Development API
   (master)       (shadow)

---

# 5. 処理フロー

1. Client が Shadow Proxy にリクエスト送信
2. Proxy が Request ID を生成
3. master / shadow に並列リクエスト
4. 両レスポンスを待機
5. JSONレスポンスを比較
6. 差分レポートを stdout に出力
7. Client には masterレスポンスを返す

---

# 6. Request ID ポリシー

Request ID は **常に Proxy が生成する**。

理由

- 信頼境界を Proxy に限定するため
- Request ID 衝突を防ぐため

ヘッダー仕様

X-Request-Id

もしクライアントが Request ID を送信した場合

X-Client-Request-Id

として保存する。

生成方式

ULID または UUID

# 9. JSON Semantic Equality

## Object

キー順は無視

{"a":1,"b":2}
{"b":2,"a":1}

は同値。

実装

key sorting

---

## Array

配列は **順序あり**

[1,2,3] != [3,2,1]

理由

- 実装が単純
- API仕様変更検出に有効

---

## null vs missing

null != missing

別の値として扱う。

# 11. Ignore Rules

動的フィールドを除外するため
JSONPath による ignore をサポートする。

例

```yaml
compare:
  ignorePaths:
    - $.timestamp
    - $.traceId
    - $.requestId
```

---

# 12. レスポンスサイズ制限

最大サイズ

10MB

超過時

result = TOO_LARGE

比較を行わない。

---

# 13. Compression Handling

レスポンスが以下の場合

Content-Encoding: gzip

Proxy は **解凍して比較する**。

理由

- JSON semantic comparison のため

---

# 14. メモリ管理

同時比較数を制限する。

デフォルト

maxConcurrentComparisons = 50

理由

10MB × 2 responses × 50
≈ 1GB memory worst case

制限超過時

result = ERROR

---

# 15. Diff 検出

差分検出時は JSONPath を出力する。

例

```json
{
  "path": "$.user.name",
  "master": "Alice",
  "shadow": "Alicia"
}
```

# 17. コンフィグ

すべての値は設定可能とする。

```yaml
proxy:

  upstream:
    master: http://existing-api:8080
    shadow: http://dev-api:8080

  timeout:
    master: 60s
    shadow: 60s

  body:
    maxBytes: 10485760

  compare:
    enabled: true
    arrayOrderSensitive: true
    nullVsMissingEqual: false

    ignorePaths:
      - $.timestamp
      - $.traceId

  concurrency:
    maxComparisons: 50

  requestId:
    header: X-Request-Id
```
